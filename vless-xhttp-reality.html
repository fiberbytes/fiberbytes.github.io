VLESS-XHTTP-Reality

DRAFT! DRAFT! DRAFT!

The WebSocket transport in Xray is deprecated and may be removed. The recommended modern alternative is XHTTP. The configuration in this tutorial was copied from https://github.com/XTLS/Xray-examples/tree/main/VLESS-XHTTP-Reality/minimal-steal_others.

Prepare server

1. SSH into your server as root:

ssh root@my.server.ip.address

2. Choose a port number. The classic choice is port 443. The port number can be changed according to the needs of your situation. If you change it, you will need to also change the port you open in step 3.

3. Open port 443 (or your choice for port) for TCP input.

4. Update your server:

apt update && apt upgrade -y

5. Install Xray:

bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install

Choose parameters

6. Choose your client UUID. The field UUID can hold any string of length 1-30 bytes. If you want to fill it with a classic UUID then generate one by executing:

xray uuid

7. Choose your secret path. Choose a path to reach Xray. It will typically be 8-16 characters and must be kept secret. In the examples it is indicated by /secretpath. Replace this by your actual choice for a secret path.

8. Generate a public-private key pair. The private key goes in the server configuration. The public key goes in the client configuration. Generate them both with a single command:

xray x25519

9. Choose the Reality target. The target field was originally named dest, but the new name has been used since version 24.10.31. The minimum requirements for the target website are:

  • It must be a foreign website.
  • It must support TLS 1.3.
  • It must support HTTP/2.
  • The domain name must not redirect (except that the bare domain may potentially redirect to the www subdomain).

You can check for TLS 1.3 on sites such as https://www.ssllabs.com/ssltest. You can check for HTTP/2 support in the Network tab of Developer Tools (F12) in Chrome or Firefox.

On Linux you can check with the command:

curl -I --tlsv1.3 --http2 https://www.example.com

The target is typically a large global site such as www.microsoft.com, www.apple.com, or www.samsung.com. It must be reachable from your location.

Some people find it works better to scan for a TLS-supporting site with a similar IP address to your server. You can scan for Reality-compatible sites with https://github.com/XTLS/RealiTLScanner.

10. For serverNames, take the Cert's allowed domains returned by executing:

xray tls ping www.example.com

11. The shortIds contains a list of shortIds available to the client. It can be used to distinguish different clients. If an empty string is specified, the client shortId can be empty. If non-empty, it should contain a hexadecimal number with an even length from 2 to 16 hexadecimal characters.

12. Choose spiderX. It is recommended that the initial path and parameters for the crawler be different for each client.

13. Choose the fingerprint. Xray uses the uTLS library to simulate a client browser with the given TLS fingerprint.

Configure server

14. Edit the Xray server configuration:

vi /usr/local/etc/xray/config.json

15. Press dG to delete existing contents to end of file.

16. Press i to enter insert mode.

17. Copy and paste contents like this, except use your parameters:

{
    "inbounds": [
        {
            "listen": "0.0.0.0",
            "port": 443,
            "protocol": "vless",
            "settings": {
                "clients": [
                    {
                        "id": "f4192bd9-1540-4af9-a233-ebc0c630c0d2", 
                        "flow": ""
                    }
                ],
                "decryption": "none"
            },
            "streamSettings": {
                "network": "xhttp",
                "xhttpSettings": {
                    "path": "/secretpath" 
                },
                "security": "reality",
                "realitySettings": {
                    "target": "www.example.com:443",
                    "serverNames": [
                        "www.example.com"
                    ],
                    "privateKey": "gPxmQMf5fsG-ze0Vitw2I-DVl9KWwaSiSaPWZJ8HSk0", 
                    "shortIds": [ 
                        "0123456789abcdef"
                    ]
                }
            },
            "sniffing": {
                "enabled": true,
                "destOverride": [
                    "http",
                    "tls",
                    "quic"
                ]
            }
        }
    ],
    "outbounds": [
        {
            "protocol": "freedom",
            "tag": "direct"
        },
        {
            "protocol": "blackhole",
            "tag": "block"
        }

    ]
}

18. Press Esc to escape insert mode.

19. Press :wq to write to file and quit the editor.

20. Restart Xray with your configuration:

systemctl restart xray

21. Check the status of Xray after your restart with the new configuration:

systemctl status xray

Configure client

22. Download the Windows 64-bit clien zip file from https://github.com/2dust/v2rayN/releases.

23. Unzip the zip file.

24. Launch v2rayN.exe.

25. Key in a VLESS configuration corresponding to this Xray core client configuration, but changing the details to match those on your server:

{
    "routing": {
        "rules": [
            {
                "ip": [
                    "geoip:private"
                ],
                "outboundTag": "direct"
            }
        ]
    },
    "inbounds": [
        {
            "listen": "127.0.0.1",
            "port": 10808,
            "protocol": "socks"
        }
    ],
    "outbounds": [
        {
            "protocol": "vless",
            "settings": {
                "address": "my.server.ip.address", 
                "port": 443, 
                "id": "f4192bd9-1540-4af9-a233-ebc0c630c0d2", 
                "encryption": "none",
                "flow": ""
            },
            "streamSettings": {
                "network": "xhttp",
                "xhttpSettings": {
                    "path": "/secretpath" 
                },
                "security": "reality",
                "realitySettings": {
                    "serverName": "www.example.com", 
                    "publicKey": "fnuLLbCV9Zj77N70tS-XTPc7UGbIt4_KWZIw1x8P2y0", 
                    "shortId": "0123456789abcdef", 
                    "spiderX": "/", 
                    "fingerprint": "chrome" 
                }
            },
            "tag": "proxy"
        },
        {
            "protocol": "freedom",
            "tag": "direct"
        },
        {
            "protocol": "blackhole",
            "tag": "block"
        }

    ]
}

26. Confirm the configuration, and start the connection from your client to your server.

27. Watch then message pane, and check that you see a positive ping time (a ping time of -1 would mean an error has occurred).

28. Configure your Windows system proxy or an individual browser to use the SOCKS proxy on 127.0.0.1 port 10808.
