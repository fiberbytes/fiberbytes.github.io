Prepare server

1. Set up DNS `A` records for your domain, which we we will put as www.example.com and example.com in the examples.

2. SSH into your server as root:

ssh root@www.example.com

3. Open port 80 and port 443 for TCP input.

4. Update your server:

apt update && apt upgrade -y

Obtain SSL certificate

5. Get the ACME script for obtaining an SSL certificate:

apt install -y socat

curl https://get.acme.sh | sh

6. Set a shorter alias for the ACME shell script:

alias acme.sh=~/.acme.sh/acme.sh

7. Set up ACME shell script auto-update:

acme.sh --upgrade --auto-upgrade

8. Change the default Certificate Authority to Let's Encrypt:

acme.sh --set-default-ca --server letsencrypt

9. Request issue of an SSL certificate:

acme.sh --issue -d www.example.com -d example.com --standalone

10. Install the certificate to the /etc/ssl/private directory:

acme.sh --install-cert -d www.example.com -d example.com --fullchain-file /etc/ssl/private/fullchain.cer --key-file /etc/ssl/private/private.key

11. Set the owner and group to work with the Xray server configuration file:

chown -R nobody:nogroup /etc/ssl/private

12. Force certificate renewal:

acme.sh --renew -d www.example.com -d example.com --force

Set up Nginx web server

13. Install Nginx:

apt install -y nginx

14. Create new Nginx available site configuration:

vi /etc/nginx/sites-available/www.example.com

15. Press i to enter insert mode.

16. Copy and paste contents like this, except with your domain instead of www.example.com and with your choice of secret path to reach Xray:

server {
	listen 443i ssl;
	listen [::]:443 ssl;
        http2 on;

	server_name www.example.com example.com;

	index index.html index.htm index.nginx-debian.html;
	root /var/www/html;

	ssl_certificate /etc/ssl/private/fullchain.cer;
	ssl_certificate_key /etc/ssl/private/private.key;
	ssl_protocols TLSv1.2 TLSv1.3;
	ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384;
	
	location /secretpath {
	if ($http_upgrade != "websocket") {
		return 404;
	}
        proxy_pass http://127.0.0.1:1234;
	proxy_redirect off;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 5d;
    }
}

17. Press Esc to escape insert mode.

18. Press :wq to write to file and quit the editor.

19. Enable the new site:

ln -s /etc/nginx/sites-available/www.example.com /etc/nginx/sites-enabled/

20. Delete the default site listening on port 80:

rm /etc/nginx/sites-enabled/default

21. Restart Nginx with all these changes:

systemctl restart nginx

22. Check the status of the Nginx web server:

systemctl status nginx

Set up Xray proxy server

23. Install Xray:

bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install

24. Generate your unqiue client UUID:

xray uuid

25. Edit the Xray server configuration:

vi /usr/local/etc/xray/config.json

26. Press dG to delete existing contents to end of file.

27. Press i to enter insert mode.

28. Copy and paste contents like this, except use your UUID and your choice of secret path:

{
  "log": {
    "loglevel": "warning"
  },
  "inbounds": [
    {
      "listen": "127.0.0.1",
      "port": 1234,
      "protocol": "vless",
      "settings": {
        "clients": [
          {
            "id": "5c632153-35c4-4e79-b337-49c73f553f3b" 
          }
        ],
        "decryption": "none"
      },
      "streamSettings": {
        "network": "ws",
        "wsSettings": {
          "path": "/secretpath" 
        }
      },
      "sniffing": {
        "enabled": true,
        "destOverride": [
          "http",
          "tls"
        ]
      }
    }
  ],
  "outbounds": [
    {
      "tag": "direct",
      "protocol": "freedom",
      "settings": {}
    },
    {
      "tag": "blocked",
      "protocol": "blackhole",
      "settings": {}
    }
  ],
  "routing": {
    "domainStrategy": "IPIfNonMatch",
    "rules": [
      {
        "ip": [
          "geoip:private"
        ],
        "outboundTag": "blocked"
      }
    ]
  }
}

29. Press Esc to escape insert mode.

30. Press :wq to write to file and quit the editor.

31. Restart Xray with your configuration:

systemctl restart xray

32. Check the status of Xray after your restart with the new configuration:

systemctl status xray

Set up v2rayN client

33. Download the Windows 64-bit clien zip file from https://github.com/2dust/v2rayN/releases.

34. Unzip the zip file.

35. Launch v2rayN.exe.

36. Key in a VLESS configuration corresponding to this Xray core client configuration, but changing the details to match those on your server:

{
  "log": {},
  "inbounds": [
    {
      "port": "10808",
      "protocol": "socks",
      "settings": {
        "auth": "noauth",
        "udp": true
      }
    },
    {
      "port": "10809",
      "protocol": "http",
      "settings": {}
    }
  ],
  "outbounds": [
    {
      "protocol": "vless",
      "settings": {
        "address": "www.example.com",
        "port": 443,
        "id": "5c632153-35c4-4e79-b337-49c73f553f3b", 
        "encryption": "none"
      },
      "streamSettings": {
        "network": "ws",
        "security": "tls",
        "wsSettings": {
          "path": "/secretpath
        },
        "tlsSettings": {
          "allowInsecure": false,
          "serverName": "www.example.com", 
          "fingerprint": "chrome" 
        }
      }
    },
    {
      "tag": "direct",
      "protocol": "freedom",
      "settings": {}
    },
    {
      "tag": "blocked",
      "protocol": "blackhole",
      "settings": {}
    }
  ],
  "routing": {
    "domainStrategy": "IPOnDemand",
    "rules": [
      {
        "ip": [
          "geoip:private"
        ],
        "outboundTag": "direct"
      }
    ]
  }
}

37. Confirm the configuration, and start the connection from your client to your server.

38. Watch then message pane, and check that you see a positive ping time (a ping time of -1 would mean an error has occurred).

39. Configure your Windows system proxy or an individual browser to use the SOCKS proxy on 127.0.0.1 port 10808.
